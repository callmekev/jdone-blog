# PKI、公鑰、私鑰是什麼？用買早餐完全聽懂加密概念（零基礎版）

你每天打開瀏覽器看到網址列的小鎖頭 🔒，其實背後就是「PKI、公鑰、私鑰」在保護你。  
這篇用「買早餐」的比喻，讓你 10 分鐘完全搞懂，不會再被這些名詞嚇到！

## 情境：你去買早餐，想安全地把錢包給老闆娘

### 問題 1：怎麼證明「你就是你」？
你跟老闆娘說：「我就是 JayQiu！」  
但老闆娘怎麼相信你不是冒牌的？

→ 這就是「身份認證」的問題

### 問題 2：怎麼安全地把 100 塊給老闆娘？
你直接把 100 塊現金遞過去 → 路上可能被搶  
你用 Line 傳紅包 → 怕被中間人偷看

→ 這就是「資料加密」的問題

PKI 就是同時解決上面兩個問題的「全世界通用的解決方案」！

## 用早餐店來比喻整個 PKI 系統

| 現實世界             | 網路世界              | 說明                                                                 |
|----------------------|-----------------------|----------------------------------------------------------------------|
| 老闆娘（網站）        | 網站伺服器            | 賣豆漿包子的就是她                                                   |
| 你（使用者）          | 你的瀏覽器            | 想買早餐的人                                                         |
| 衛生局（可信任的第三方）| 證書機構（CA）        | 大家相信衛生局蓋的章是真的                                           |
| 你的身份證            | 數位證書（Certificate）| 上面有你的名字 + 公鑰 + 衛生局的簽名                                 |
| 公鑰（可以給所有人）  | 像鎖頭 🔓             | 任何人都可以用這個鎖頭把東西鎖起來，但只有你能打開                   |
| 私鑰（絕對不能給人）  | 像鑰匙 🔑             | 只有你有，永遠放在自己口袋                                           |

## 實際流程：你去買早餐的加密版

1. **老闆娘先去衛生局申請營業登記**  
   → 網站向 Let’s Encrypt（免費的證書機構）申請數位證書  
   → 證書裡面包含「老闆娘的公鑰」+「Let’s Encrypt 的簽章」

2. **你走進早餐店，老闆娘把證書遞給你**  
   → 你的瀏覽器收到網站的數位證書  
   → 瀏覽器檢查：「Let’s Encrypt 的簽章是真的嗎？」→ 是！  
   → 代表「這真的是老闆娘，不是冒牌貨」

3. **你用老闆娘的公鑰（鎖頭）把訂單鎖起來傳給她**  
   → 瀏覽器產生一組「對稱金鑰」（像現金），用老闆娘的公鑰加密後傳過去  
   → 路上就算被搶也打不開（因為沒有私鑰）

4. **老闆娘用私鑰（鑰匙）打開，拿到你的訂單**  
   → 之後你們用這組對稱金鑰聊天（速度快又安全）

整個過程就是 HTTPS 的「TLS 握手」！

## 超簡單記憶口訣

- **公鑰** = 鎖頭 🔓 → 可以給全世界  
- **私鑰** = 鑰匙 🔑 → 永遠藏好，只有一把  
- **數位證書** = 身份證 + 鎖頭 + 衛生局蓋章  
- **CA（證書機構）** = 衛生局，大家都相信它蓋的章是真的

## 常見問題一次解答

**Q：私鑰真的只有一台伺服器有嗎？**  
A：對！就像老闆娘把鑰匙藏在家裡保險箱，絕對不會給別人。

**Q：如果有人偷了私鑰怎麼辦？**  
A：就跟老闆娘家被偷一樣慘！所以大公司會用 HSM（硬體安全模組）保護私鑰。

**Q：為什麼現在很多網站都是 Let’s Encrypt？**  
A：因為它免費、自動續期，就像衛生局現在免費幫大家蓋章一樣。
